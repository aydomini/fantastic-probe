## [4.1.9] - 2026-02-01

### 改进

- 🔍 **增强 JSON 验证和诊断**
  - 分三步验证 `language_tags_json`：空值检查、JSON 格式验证、紧凑化重格式化

  - 添加详细诊断日志：记录 JSON 长度和内容（前 200 字符）
  - 任何验证失败时都使用默认值，确保不会传递无效 JSON 给 jq --argjson

## [4.1.8] - 2026-02-01

### 修复

- 🔧 **修复 jq --argjson 参数错误**
  - **根本原因**：`language_tags_json` 可能包含无效的 JSON 格式
  - **错误表现**：`jq: invalid JSON text passed to --argjson`
  - **解决方案**：
    - 调用 `convert_to_emby_format` 前验证 `language_tags_json` 是否为有效 JSON
    - 如果无效，使用默认值 `{"audio_languages":[],"subtitle_languages":[],"chapters":0}`
    - 添加详细的错误日志，显示原始无效值

### 改进

- 📊 **增强 Python 脚本错误诊断**
  - Python JSON 输出改为紧凑格式：`separators=(',', ':')`
  - 使用 `printf` 代替 `echo` 传递数据，避免字符转义问题
  - 捕获 Python stderr 到临时文件，失败时显示错误详情
  - 语言标签解析失败时显示原始输出，便于诊断

## [4.1.3] - 2026-02-01

### 修复

- 🔧 **修复关键 Bug：bd_list_titles 输出解析失败**
  - **根本原因**：Bash 中管道 + heredoc 冲突，导致 Python `sys.stdin.read()` 读到空字符串
  - **影响**：所有蓝光 ISO 的语言标签全部显示 0 音频、0 字幕、0 章节
  - **解决方案**：改用临时 Python 脚本文件 + 管道传递数据
  - 现在能正确提取音频/字幕语言标签

- 🔧 修复 bd_list_titles 调试日志被 DEBUG 模式静默的问题
  - 将关键诊断日志从 `log_debug` 改为 `log_info`
  - 现在即使不启用 DEBUG 模式，也能看到 bd_list_titles 输出和"非蓝光目录"提示

- 🔧 **修复比特率不显示问题**
  - **原因**：bluray: 协议访问 ISO 时，ffprobe 不返回 `format.bit_rate`
  - **解决方案**：当 `format.bit_rate` 为 null 时，从 ISO 文件大小和时长自动计算
  - 计算公式：`bitrate = (文件大小 * 8) / 时长`
  - 确保 Emby 能正确显示蓝光 ISO 的总比特率

### 增强

- 🛡️ 增强 ISO 挂载健壮性
  - 启动时自动清理残留的挂载点
  - 卸载失败时自动重试（最多 3 次）
  - 重试失败后强制卸载（`umount -f`）
  - 验证卸载结果，防止挂载点泄漏

- 📋 改进日志输出
  - 显示 bd_list_titles 输出前 5 行（用于诊断）
  - 显示具体挂载点路径
  - 详细的错误处理和退出码记录

- ⏱️ 优化 Cron 扫描间隔
  - 从每 3 分钟扫描改为每 1 分钟扫描
  - 更快速地发现和处理新文件

### 工具

- 🧪 新增独立测试脚本 `test-bd-parse.sh`
  - 用于验证 bd_list_titles 语言标签提取逻辑
  - 独立运行，不依赖项目安装
  - 详细的诊断输出和统计信息
